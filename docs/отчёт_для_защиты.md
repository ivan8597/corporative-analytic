## 1. Концептуальное описание

### Бизнес-контекст

Корпоративный заказчик **ООО ТехноСофт** приобрёл доступ к платформе IT Resume для обучения **15 сотрудников**. Руководство и HR не имеют единой картины: кто учится, где возникают сложности, есть ли отдача от вложений в обучение.

### Проблема

- Нет прозрачности по **прогрессу** и **вовлечённости** сотрудников.
- Непонятно, **на каком этапе курса** происходит отвал.
- Финансовый директор не может оценить **ROI** обучения.
- Методисты не видят **проблемные задачи** в контенте.

### Цель проекта

Построить дашборд в Metabase, который отвечает на вопросы **5 групп стейкхолдеров** и позволяет за 30 секунд понять общую картину по клиенту.

### Scope

| Параметр | Значение |
|----------|----------|
| Клиент | id = 1 (ООО ТехноСофт) |
| Период | октябрь 2025 — май 2026 |
| Источник данных | PostgreSQL (`it_resume_analytics`) |
| Таблицы | users, student_progress, tasks, attempts, modules, clients |

### Ограничения

- Данные **синтетические** (демо-стенд для учебного проекта).
- Метрики **NPS** и **реальный revenue** в БД отсутствуют — использованы прокси: retention, completion rate, активность.
- Бенчмарк по всем клиентам основан на **5 корпоративных заказчиках** в датасете.

## 3. Содержательная часть (пример разбора метрики)

### Метрика: Конверсия по этапам курса (воронка)

#### Гипотеза

Студенты «застревают» на переходе от модуля Python к модулю SQL из-за резкого роста сложности.

#### SQL-запрос

Файл: `sql/metrics/02_course_funnel.sql`

```sql
WITH client_students AS (
    SELECT id AS user_id FROM users
    WHERE client_id = {{client_id}} AND role = 'student'
),
funnel AS (
    SELECT 'Зарегистрированы' AS stage, 1 AS stage_order, COUNT(*) AS students
    FROM client_students
    UNION ALL
    SELECT 'Начали обучение', 2, COUNT(DISTINCT a.user_id)
    FROM attempts a JOIN client_students cs ON cs.user_id = a.user_id
    -- ... этапы по модулям ...
)
SELECT stage, students, conversion_pct FROM funnel ORDER BY stage_order;
```

**Логика:** считаем уникальных студентов на каждом этапе; конверсия — от базы «Зарегистрированы».

#### Визуализация

- Тип: **Funnel chart**
- Название: «Конверсия по этапам курса»
- Фильтр: `client_id = 1`

#### Результаты

| Этап | Студентов | Конверсия |
|------|-----------|-----------|
| Зарегистрированы | 15 | 100% |
| Начали обучение | 15 | 100% |
| Завершили модуль 1 (Python) | 15 | 100% |
| Завершили модуль 2 (SQL) | 7 | 46,7% |
| Завершили курс | 0 | 0% |

#### Вывод

Главное «узкое горло» — **модуль 2 (SQL)**: потеря 53% студентов. До финала курса не дошёл никто.

#### Рекомендация

- Добавить мостовой блок «SQL для Python-разработчиков».
- Office hours куратора 2 недели после старта SQL.
- KPI: ≥ 70% прохождения модуля 2 за 30 дней.

## 5. Финальные выводы и рекомендации

### Ключевые выводы

1. **Вовлечённость падает** после ноября 2025 — обучение не закреплено в рабочем процессе.
2. **Контент модуля SQL** — главный барьер, не мотивация на входе.
3. **Проблема масштабируется** на всех корпоративных клиентов (0% финального completion).
4. **HR-риск:** все 15 сотрудников формально в статусе «Отстаёт».

### План действий

| # | Действие | Ответственный | Срок |
|---|----------|---------------|------|
| 1 | Re-engagement: напоминания + слоты обучения | HR + куратор | 1 неделя |
| 2 | Разбор и доработка модуля SQL | Методист | 2 недели |
| 3 | 1:1 с 5 студентами (progress <40%) | Куратор | 2 недели |
| 4 | Custdev с 3 корпоративными клиентами | Продукт | 1 месяц |
| 5 | Новая метрика «завершили модуль N» на дашборде | Аналитик | 1 спринт |

### Итог для устной защиты (30 секунд)

> По клиенту ООО ТехноСофт все 15 сотрудников начали и прошли Python, но только 7 дошли до SQL, а курс целиком не завершил никто. Причины — падение активности после ноября и высокая сложность SQL/API-задач. Рекомендую: re-engagement, доработку модуля 2 и персональную работу куратора. Дашборд в Metabase позволяет отслеживать прогресс в реальном времени с фильтрами по клиенту и периоду.
